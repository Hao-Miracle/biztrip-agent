收集了用户最常问的问题，快速找到答案。

## 🚀 使用相关

### Q: 必须配置 LLM API Key 才能用吗？

**不用。** 不配 API Key 也能完整使用，走纯规则引擎，功能完全够用。LLM 是锦上添花的增强选项。

### Q: 支持哪些邮箱？

目前测试并支持：QQ 邮箱、163 邮箱、126 邮箱、Gmail、Outlook/Hotmail。

理论上支持所有标准 IMAP 协议的邮箱，可在 `.env` 中手动指定 IMAP 服务器地址。

### Q: 扫描多少封邮件？

默认扫描最近 60 封。可以通过 `--count` 参数调整，或按日期范围筛选。

### Q: 会修改我邮箱里的邮件吗？

**绝对不会。** 我们只有 IMAP 只读权限，只读取不修改、不删除、不发送。

## 💰 数据准确相关

### Q: 提取的金额准确吗？

我们对金额准确性非常重视：
- 优先从 PDF 文件名提取（最可靠）
- 其次是 PDF 文本中的"价税合计"
- 然后是邮件正文的"合计 XX 元"
- 最后才是 ¥ 符号兜底

但平台格式千变万化，建议**生成后人工核对一遍**，确保万无一失。

### Q: 发票可以验真吗？

不做发票验真。我们只做归集和整理，把发票原件归档好，验真请通过官方渠道。

### Q: 识别错了怎么办？

欢迎提交 Issue 告诉我们哪个平台、哪种邮件识别错了。我们会不断优化规则。

## 🔧 技术相关

### Q: 支持 Windows 吗？

支持。Python 跨平台，Windows / macOS / Linux 都能运行。

### Q: 可以用本地模型吗？

可以。只要兼容 OpenAI 协议（如 Ollama、LM Studio 等），在 `.env` 中配置 `LLM_BASE_URL` 指向本地地址即可。

### Q: 怎么新增平台支持？

两种方式：
1. 自己改代码：在 `phase1/classify_emails.py` 的 `DOMAIN_RULES` 加域名，在 `phase1/extract_emails.py` 加提取逻辑
2. 提 Issue：把邮件转发给我们（脱敏后），我们来加

### Q: 数据存在哪里？

全部在你本地 `output/` 目录下：
- `output/差旅汇总_*.xlsx` — Excel 报表
- `output/附件/` — 原始 PDF/ZIP 附件

不上传任何服务器，不存云端。
