# 功能范围

## MVP 功能清单

### 输入
- Gmail / QQ 邮箱 / 163 邮箱 OAuth 授权登录
- 扫描最近 60 天收件箱
- 自动筛选出差相关邮件

### 核心处理
- 全品类识别和分类（机票/火车票/大巴票/公交票/网约车/酒店/油费单/发票）
- 每类文档的完整结构化提取（日期、金额、供应商、出行人、出发地/目的地等）
- 出差行程聚合（同一时间段+同一目的地归并为一次出差）
- 自动下载邮件附件作为原件存档

### 输出
- **出差行程卡片**：按出差行程聚合的日程视图，出差中查阅
- **报销汇总表**：按出差行程汇总费用，生成可提交的报销明细
- **Excel 导出**：出差行程总表 + 各类目明细 + 报销汇总，按类目分 Sheet
- **原件下载**：每条记录可查看/下载原始订单文件

### 信任
- 隐私说明页（处理完即删原始邮件）

## MVP 不做

| 不做的功能 | 原因 |
|-----------|------|
| 用户注册系统 | 直接 OAuth 登录，不需要额外注册 |
| 移动端适配 | MVP 只适配电脑浏览器 |
| 企业对接/集成 | 不接入任何企业系统 |
| 多语言 | 中文优先 |
| 分享/协作 | 后期 |
| 连续扫描/监控 | MVP 只做用户主动触发的一次性扫描 |
| 发票验真 | 归集原件交给用户，不做验真 |
| 图片上传 | 所有出差票据都有邮件渠道 |

## 产品进化路径

```text
Phase 1（工作流验证）            Phase 2（Web 应用）
用户自己配置并跑工作流           用户在网页上授权邮箱
→ AI 提取信息                    → AI 提取信息
→ 生成 Excel                     → 浏览器查看行程卡片 + 导出 Excel
→ 自己验证识别准确率              → 用户反馈修正 → 系统迭代
```
