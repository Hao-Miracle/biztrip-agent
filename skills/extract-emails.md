# Skill: 信息提取 · Email Extract

从已分类的差旅邮件中提取结构化字段（金额、日期、路线、订单号等）。包含 8 个平台专用解析器和通用正则兜底。

## 触发条件

用户需要对已确认差旅相关的邮件提取金额、日期、行程等结构化数据。

## 调用方式

```bash
python3 phase1/extract_emails.py
```

## 平台专用解析器

| 解析器 | 平台 | 提取字段 |
|------|------|------|
| `parse_qunar_flight` | 去哪儿网 | 金额, 订单号, 日期, 出发地/目的地 |
| `parse_didi` | 滴滴出行 | 金额, 日期, 出发地/目的地 |
| `parse_gaode` | 高德打车 | 金额, 日期 |
| `parse_12306` | 12306 | 金额, 日期, 车次 |
| `parse_huazhu` | 华住酒店 | 金额, 日期, 酒店名称, 发票号码 |
| `parse_crestv` | 智慧发票 | 金额, 商家, 品类(油费/餐饮) |
| `parse_txffp` | 票根 | 金额, 日期, 通行费类型 |
| `parse_ctrip_flight` | 携程 | 金额, 日期, 出发地/目的地 |

未匹配的平台自动走 `generic_extract()` 兜底。

## 金额提取优先级（全模块统一）

```
1. PDF 文件名中的金额（如 1755.00.pdf）
2. PDF 文本中的「价税合计(小写)」
3. 正文中的「合计...元」
4. 正文中的 ¥/￥ 符号
5. 正文中的「XX.XX 元」
6. 兜底：正文中所有 XX.XX 数值最大值
```

**禁止取税额**：`¥` 模式放在最后，避免匹配到发票税额。

## PDF 附件

- 使用 PyPDF2 提取 PDF 文本
- 追加到邮件正文末尾参与提取
- 需要依赖：`PyPDF2`

## 输出

每条邮件一个字典：
```python
{'金额': float|'', '日期': str|'', '出发地': str|'', '目的地': str|'', '订单号': str|'', ...}
```
