# 需求探索过程 — 从 AI 副业想法到 BizTrip Agent

2026-06-29，经过多轮探索和方向验证，最终确定了 BizTrip Agent 的产品方向。

## 初始背景

- 目标：AI 副业/个人项目，持续收入
- 资源：AI 工具（Hermes、LLM、自动化工作流）+ 旅游行业经验
- 时间：每天约 2 小时碎片时间
- 产品形态：工具型，给个人用，通用场景

## 讨论过程

### 方向探索与排除

1. **AI 内容营销/自媒体** → 排除：类似电商模式，大厂在做
2. **中小企业 AI 工作流服务** → 排除：需要对接客户，时间不允许
3. **旅行 AI 行程规划** → 排除：ChatGPT/豆包可直接替代
4. **避坑/评价工具** → 排除：法律风险（诽谤/诉讼）
5. **旅行确认信息自动归集** → 初步认可，但需解决多人拆分等需求
6. 调研发现 Trips Flow 已做了类似产品但 2 年没做起来 → 原因分析：使用频率太低（年 2-3 次）
7. **商务出差场景** → 保留。调研发现个人出差报销是市场空白

### 关键转折

- 需求分析从"急于出点子"转向第一性原理拆解
- 参考成功产品案例，理解痛点匹配模式
- 参考 Anthropic《构建有效 AI Agent》指导工作流/agent 构建思路
- 最终确定方向：出差报销助手，Phase 1 工作流验证 → Phase 2 Web 应用

## 关键结论

| 结论 | 说明 |
|------|------|
| 出差场景优于旅行 | 出差频率远超旅行，报销是刚需 |
| 个人视角是空白 | 企业工具太复杂，旅行工具太轻 |
| 工作流先行 | Phase 1 验证核心逻辑，Phase 2 再构建 Web 应用 |
| 双模式 | 出差和旅行共用引擎，输出模式不同 |
