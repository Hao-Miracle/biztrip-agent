# 差旅助手（TripDoc Agent）— 完整项目方案

> 个人出差和旅行的行程与报销信息自动归集工具。
> 用户授权邮箱后，AI 自动扫描确认邮件，提取结构化信息。行程中随时看安排，行程后一键生成报销表或日程表。

---

## 一、产品概述

### 一句话
用户授权邮箱，AI 自动识别一切行程相关的邮件（机票、火车票、酒店、门票、网约车、发票、油费单等），提取关键信息，一次扫描出两种价值。

### 核心价值（并行）

| 阶段 | 用户需要什么 | 产品做什么 |
|------|-------------|-----------|
| **行程中** | 快速查今晚住哪、明天几点的车、航班号、当地交通 | 行程卡片视图，按行程聚合，每天一张清晰安排 |
| **行程后** | 出差：整理订单和发票生成报销单 / 旅行：整理行程记录 | 按行程自动汇总，出差出报销表，旅行出行程表 |

### 两个场景，同一套引擎

| 场景 | 输入 | 输出 |
|------|------|------|
| **出差模式** | 机票、火车票、酒店、网约车、油费单、发票等 | 行程卡片 + 报销汇总表 + Excel |
| **旅行模式** | 机票、火车票、酒店、门票、租车等 | 行程卡片 + 旅行日程表 + Excel |

**核心洞察**：出差和旅行的底层逻辑完全一样——邮箱里都有确认订单，都需要 AI 自动识别、提取、归类。唯一区别在于输出：出差涉及发票需要报销表，旅行不需要。同一套引擎，切换输出模式即可。

### 目标用户
所有需要出差或旅行的个人用户——从员工到老总，从自由行到家庭游，不需要企业部署。

### 产品形态
- **Phase 1（当前）**：自动化工作流验证（用 Hermes/自动化工具搭建核心逻辑，自己用真实场景验证）
- **Phase 2（待定）**：核心逻辑包装成 Web 应用，开放给他人使用

## 二、MVP 范围

### 要做

| 功能 | 说明 |
|------|------|
| 三大邮箱 OAuth 登录 | Gmail / QQ 邮箱 / 163 邮箱，只读权限，用户自己授权 |
| 邮件扫描 | 扫描最近 60 天收件箱，自动筛选行程相关邮件 |
| 全品类识别 | 机票、火车票、大巴票、公交票、网约车、酒店、门票、租车、油费单、发票等 |
| 结构化文档提取 | 每份文档按品类提取完整结构化信息（日期、金额、供应商、出行人、出发地/目的地等） |
| 原件存档 | 自动下载邮件附件（PDF 行程单、发票等），关联存档，随时下载 |
| 行程卡片 | 按行程聚合的日程视图，出差中/旅行中查阅当天安排 |
| 报销/日程汇总 | 出差模式：自动生成报销明细表；旅行模式：自动生成旅行日程表 |
| Excel 导出 | 多 Sheet：行程总表 + 各类目明细 + 报销/日程汇总 |
| 原始文件下载 | 每条记录可查看/下载原始订单文件 |
| 隐私说明页 | 处理完即删原始邮件，仅存放提取后的结构化数据 |

### 不做

| 功能 | 原因 |
|------|------|
| 用户注册系统 | 直接 OAuth 登录即可 |
| 移动端适配 | MVP 阶段只适配电脑浏览器 |
| 企业对接/集成 | 不做企业级功能 |
| 多语言 | 中文优先 |
| 分享/协作 | 后期再考虑 |
| 连续扫描/监控 | MVP 只做用户主动触发的一次性扫描 |
| 发票验真 | 归集原件交给用户，不做验真 |
| 图片上传 | 所有票据都有邮件渠道 |

## 三、工作流设计（基于 Anthropic Agent 架构模式）

### 设计原则
> 从最简单的方案开始，只在必要时增加复杂度。—— Anthropic

### Level 1：提示链 + 路由（MVP）

```text
[Step 1] 获取邮件
  IMAP 连接 → 获取最近 60 天邮件
  → Gate: 连接成功？失败则报错退出

[Step 2] 路由分类（对每封邮件）
  LLM 判断：
    ├── 与行程不相关 → 跳过
    ├── 机票 → 专用提取 prompt
    ├── 火车票 → 专用提取 prompt
    ├── 酒店 → 专用提取 prompt
    ├── 门票/租车 → 专用提取 prompt
    ├── 网约车 → 专用提取 prompt
    ├── 油费单/发票 → 专用提取 prompt
  → Gate: 至少提取到一条有效记录？

[Step 3] 行程聚合
  LLM 分析所有提取记录，按"同一时间段 + 同一目的地"归并为一次行程

[Step 4] 生成输出
  选择模式：
  ├── 出差模式 → 行程卡片 + 报销汇总表 + Excel
  └── 旅行模式 → 行程卡片 + 旅行日程表 + Excel

[Step 5] 下载原件
  自动下载邮件附件，关联到对应行程
```

### Level 2：并行化（优化）
邮件量大时分批并行处理，降低等待时间。

### Level 3：评估器-优化器（Phase 2）
用户可手动修正提取结果 → 系统学习修正模式 → 后续提取准确率提升。

## 四、Excel 模板结构

| Sheet | 内容 |
|-------|------|
| 行程总表 | 行程编号、出行人、目的地、起止日期、天数、订单数、总金额 |
| 机票明细 | 出行人、日期、航班、出发→到达、时间、金额、订单号、原件 |
| 火车票明细 | 乘车人、日期、车次、出发站→到达站、座位等级、金额 |
| 酒店明细 | 入住人、酒店、房型、入住→离店、间数、金额 |
| 门票明细 | 使用人、景点、日期、数量、金额 |
| 网约车明细 | 乘车人、日期、路线、金额 |
| 发票/油费明细 | 日期、项目、类别、金额、供应商 |
| 报销汇总（出差模式）| 费用项目、金额、出差编号 |
| 行程/日程汇总（旅行模式）| 每日安排、景点、交通方式 |

## 五、竞品分析

| 竞品 | 定位 | 和差旅助手的差异 |
|------|------|----------------|
| TripIt | 国际个人行程管理，手动为主，不支持中文 | AI 自动提取 + 报销功能 + 双模式 |
| Trips Flow | 国内旅行行程管理，iOS only，2 年没做起来 | 出差+旅行双模式 + Web 端 + Excel 输出 |
| 每刻报销 / 合思商旅 | 企业级报销系统，需公司采购 | 个人使用，无审批流，不需企业部署 |
| 携程商旅 | 企业商旅一体化平台 | 个人视角轻量工具 |
| 钉钉AI差旅 | 企业内开通的差旅服务 | 个人独立使用，不依赖办公平台 |

**核心差异**：现有工具要么是企业视角（审批流、预算管控），要么是纯旅行视角。差旅助手两者兼顾，出差和旅行双模式，同一套引擎。

## 六、技术栈

| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | Next.js (React) | 前后端合一 |
| 样式 | Tailwind CSS | 上手快 |
| 后端 | Next.js API Routes | 同一项目 |
| 数据库 | SQLite / PostgreSQL（免费版）| 存储结构化数据 |
| 邮箱 API | Gmail API + QQ邮箱API + 163邮箱IMAP | 只读权限 |
| AI 识别 | DeepSeek-V3 API | 中文好、价格低 |
| Excel 生成 | SheetJS | 多维表格 |
| 文件存储 | Cloudflare R2 / Vercel Blob | 原件附件 |
| 部署 | Vercel（免费）| Next.js 原生平台 |

## 八、变现逻辑

**Freemium：**

| 层级 | 功能 | 定价 |
|------|------|------|
| 免费 | 单次扫描 + 导出报表（限 3 次/月）| 0 元 |
| Pro | 不限扫描、多邮箱、历史行程存档、年度汇总 | 19 元/月 或 99 元/年 |

**付费动机：**
- 月月出差的人，19 元省下每次出差后 1-2 小时的整理时间
- 年底个人汇算时，全年的出差报销汇总有价值
- 每年旅行 2 次以上的人，19 元买个省心

## 十一、风险矩阵

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 做一半不想做了 | 中 | 高 | 碎片时间推进，不追求一次性完整 |
